# バー設置シミュレーション

## 概要
このプロジェクトは、斜面ステージ上にバーを設置する物理シミュレーションです。バーの設置成功率を分析し、最適な設置条件を探索するためのツールを提供します。

**🚀 最新情報: AWS最適化版がリリースされました！**
- **8倍の高速化**を実現
- CPU使用率を85-95%に向上
- 処理時間とコストを87%削減

詳細は [IMPLEMENTATION_SUMMARY.md](IMPLEMENTATION_SUMMARY.md) をご覧ください。

## 技術仕様
- **物理エンジン**: pymunk (2D物理シミュレーション)
- **描画**: pygame (リアルタイム可視化)
- **データ分析**: pandas, numpy, matplotlib (結果分析・可視化)
- **並列処理**: multiprocessing (高速化)
- **AWS最適化**: CPU affinity, 動的チャンクサイズ, リアルタイム進捗管理

## 実行モード

### 1. INTERACTIVE（インタラクティブモード）
リアルタイムでパラメータを調整しながら結果を確認できるモード

### 2. BATCH（バッチモード）
複数の条件を自動で試行し、結果をファイル出力するモード

### 3. BATCH_PARALLEL（並列バッチモード）⭐推奨
マルチコアCPUを活用した高速バッチ処理モード

**最適化機能:**
- CPU Affinity（CPUコア固定）による高速化
- 動的チャンクサイズ計算によるロードバランシング
- リアルタイム進捗表示とメモリ監視
- グレースフルシャットダウン（中断時もデータ保存）

**使用方法:**
```bash
MODE=BATCH_PARALLEL python3 main.py
```

### 4. SINGLE（単一条件モード）
単一条件の初期/最終状態を画像出力するモード

## 入力パラメータ

| パラメータ | 変数名 | デフォルト値 | 説明 |
|-----------|--------|-------------|------|

| 項目 | 値 | 単位 | 備考 |

## 出力データ

### 成功判定基準
| 項目 | 基準値 | 単位 | 説明 |
|------|--------|------|------|
| 最大速度 | 1.0 | m/s | バーの線形速度がこの値以下 |
| 最大角速度 | 0.5 | rad/s | バーの角速度がこの値以下 |
| 角度許容範囲 | 2.0° | 度 (°) | 理想角度からの許容誤差 |
| 最小安定時間 | 1.0 | 秒 (s) | 安定状態の維持時間 |
| 位置許容誤差 | 3.0 | ピクセル (px) | 理想位置からの許容誤差 |

### 失敗判定
- **床接触**: バーが床面に接触した場合
- **複数短冊接触**: 短手方向でステージ壁面に2回以上接触
- **不安定**: 成功基準を満たさない動作状態
- **角度不正**: 最終角度が許容範囲外
- **未接触**: ステージとの接触が不十分

### CSV出力データ（バッチモード）
| 列名 | データ型 | 説明 |
|------|----------|------|
| `angle` | float | ステージ角度（度） |
| `release_x_offset` | int | リリース位置Xオフセット |
| `release_y_offset` | int | リリース位置Yオフセット |
| `relative_angle` | int | バー相対角度（度） |
| `success_rate` | float | 成功率（%） |
| `ideal_x`, `ideal_y` | float | 理想設置位置座標 |
| `avg_final_x`, `avg_final_y` | float | 平均最終位置座標 |
| `difference_from_ideal_px` | float | 理想位置からの誤差（ピクセル） |
| `difference_from_ideal_mm` | float | 理想位置からの誤差（mm） |
| `avg_short_contacts` | float | 平均短冊方向接触回数 |
| `max_short_contacts` | int | 最大短冊方向接触回数 |
| `failures_floor` | int | 床接触による失敗回数 |
| `failures_multi_short` | int | 複数短冊接触による失敗回数 |
| `failures_unstable` | int | 不安定による失敗回数 |
| `failures_angle` | int | 角度不正による失敗回数 |
| `failures_no_contact` | int | 未接触による失敗回数 |
| `failures_invalid_pos` | int | 不正初期位置による失敗回数 |
| `x_offset_std` | float | X位置バラツキの標準偏差 |
| `y_offset_std` | float | Y位置バラツキの標準偏差 |
| `angle_offset_std` | float | 角度バラツキの標準偏差 |

## UI表示位置関係

### 座標系および基準点
| 項目 | 値 | 単位 | 説明 |
|------|----|----|------|
| 画面サイズ | 800 × 600 | ピクセル | シミュレーション画面の全体サイズ |
| PPM (Pixels Per Meter) | 100.0 | px/m | 物理単位とピクセルの変換係数 |
| ステージ基点座標 | (400, 400) | ピクセル | ステージの回転中心点 (BASE_X, BASE_Y) |
| ステージ長さ | 400 | ピクセル | ステージの斜面部分の長さ |
| 壁面高さ | 100 | ピクセル | ステージの壁面部分の高さ |

### サイズ変換式とスケール比較
**物理単位 ⇔ ピクセル変換式：**
- **ピクセル → メートル**: `メートル = ピクセル ÷ PPM`
- **メートル → ピクセル**: `ピクセル = メートル × PPM`

**バーとステージのサイズ比較：**
| 要素 | 物理サイズ | ピクセルサイズ | ステージとの比較 |
|------|-----------|--------------|----------------|
| **バー長手方向** | 1.0 m | 100 px | ステージ斜面の 1/4 |
| **バー短手方向** | 0.2 m | 20 px | ステージ斜面の 1/20 |
| **ステージ斜面** | 4.0 m | 400 px | バー長手の 4倍 |
| **ステージ壁面** | 1.0 m | 100 px | バー長手と同じ |

**スケール感の参考：**
- バーの長手方向（100px）はステージ斜面（400px）の25%
- バーの短手方向（20px）はステージ斜面（400px）の5%
- ステージ壁面（100px）はバーの長手方向と同じ長さ

### リリース位置の計算方法
リリース座標は以下の式で計算されます：
- **実際のX座標** = BASE_X (400) + `release_x_offset`
- **実際のY座標** = BASE_Y (400) - `release_y_offset`

### UI上の表示要素
| 表示要素 | 色 | 位置 | 説明 |
|----------|----|----|------|
| バー（通常時） | 青色 | シミュレーション位置 | 物理シミュレーション中のバー |
| バー（床接触時） | 赤色（点滅） | 接触位置 | 床に接触した瞬間のバー |
| リリース位置マーカー | 赤色（正常）/紫色（不正） | リリース座標 | バーの初期投下位置 |

| 誤差線 | 緑色/橙色 | 理想位置～実際位置 | 位置誤差の可視化 |
| 座標グリッド | 薄灰色 | 全画面 | 100ピクセル間隔の座標参考線 |

### インタラクティブモードの操作表示位置
| 情報 | 表示位置 | 内容 |
|------|---------|------|
| パラメータ調整 | 画面左上 | ステージ角度、リリース位置、相対角度 |

| アラート表示 | 画面中央 | 床接触、複数短冊接触の警告 |

## 使用方法

### 1. 基本実行
```python
# main.py の MODE を変更
MODE = "INTERACTIVE"  # または "BATCH", "BATCH_PARALLEL", "SINGLE"
python main.py
```

### 2. パラメータ調整
摩擦設定の例：
```python
BAR_FRICTION = 2.0      # 高摩擦に設定
WALL_FRICTION = 2.0     # 高摩擦に設定
```

### 3. バッチ範囲調整
```python
BATCH_PARAM_RANGES = {
    'angle': range(15, 51, 5),           # 15°から50°まで5°刻み
    'release_x_offset': range(-70, 10, 1), # X位置範囲
    'release_y_offset': range(40, 110, 1), # Y位置範囲
    'relative_angle': [0]                 # 相対角度固定
}
```
